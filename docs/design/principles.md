# Design Principles

## Calm And Focused

The interface should support breathwork without visual noise. The breathing coach is the primary experience, especially during an active session.

## Minimal By Default

Show only the controls needed for the user's current context. Hide secondary workflows during breathwork sessions.

## Session Priority

When a breathwork session starts, the breathing coach should take the available screen real estate while keeping the header visible.

## Accessible First

The user interface must respect WCAG accessibility standards. Contrast, focus states, touch targets, readable typography, reduced motion, and assistive labels are design requirements.

## Touch First

The app should work comfortably on mobile. Critical controls must remain visible, reachable, and separated from content.

## Token Driven

Use Chakra UI tokens, semantic tokens, text styles, layer styles, and recipes instead of scattered raw CSS values.

## Consistent Components

Repeated controls should use consistent visual treatment. Variations should come from defined component variants, not one-off styling.
